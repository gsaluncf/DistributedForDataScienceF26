AdFlow Command Reference Sheet

A consolidated lookup of CLI commands, SAM patterns, and Console paths used in this project. Everything here builds on what you have done in weeks 1-4. Replace YOURID with your chosen student ID.

SAM (Serverless Application Model)

What Command
Build the project sam build
Deploy (first time, interactive) sam deploy --guided --stack-name adflow-YOURID
Deploy (subsequent, uses saved config) sam deploy
Invoke Lambda locally (for debugging) sam local invoke WorkerFunction -e event.json
View stack outputs aws cloudformation describe-stacks --stack-name adflow-YOURID --query "Stacks[0].Outputs"
Tear down everything sam delete --stack-name adflow-YOURID

SQS (Simple Queue Service)

What Command
Get queue URL aws sqs get-queue-url --queue-name adflow-YOURID-input
Check message count aws sqs get-queue-attributes --queue-url QUEUE_URL --attribute-names ApproximateNumberOfMessages
Send a test message aws sqs send-message --queue-url QUEUE_URL --message-body '{"test": true}'
Receive messages (peek) aws sqs receive-message --queue-url QUEUE_URL --max-number-of-messages 5
Purge all messages aws sqs purge-queue --queue-url QUEUE_URL
List all queues (filtered) aws sqs list-queues --queue-name-prefix adflow-YOURID

SQS in Python (boto3)

import boto3, json

sqs = boto3.client("sqs")

# Send a message
sqs.send_message(
    QueueUrl="https://sqs.us-east-1.amazonaws.com/123456/adflow-YOURID-input",
    MessageBody=json.dumps({"opportunity_id": "test-001", ...}),
)

# Receive messages
response = sqs.receive_message(QueueUrl=QUEUE_URL, MaxNumberOfMessages=10)
for msg in response.get("Messages", []):
    body = json.loads(msg["Body"])
    # process...
    sqs.delete_message(QueueUrl=QUEUE_URL, ReceiptHandle=msg["ReceiptHandle"])

DynamoDB in Python (boto3)

import boto3
from decimal import Decimal

dynamodb = boto3.resource("dynamodb")
table = dynamodb.Table("adflow-YOURID-results")

# Write an item
table.put_item(Item={
    "opportunity_id": "test-001",
    "winning_advertiser_id": "adv_001",
    "winning_bid_amount": Decimal("3.50"),
    "winning_score": Decimal("6.7375"),
    "score_margin": Decimal("0.9625"),
    "processed_at": "2025-03-10T20:15:01Z",
})

# Read an item
item = table.get_item(Key={"opportunity_id": "test-001"})["Item"]

# Scan all items into a list
items = table.scan()["Items"]

# Load into a Pandas DataFrame
import pandas as pd
df = pd.DataFrame(items)
df["winning_bid_amount"] = df["winning_bid_amount"].astype(float)

Remember: DynamoDB stores numbers as Decimal, not float. Convert with Decimal(str(value)) when writing and .astype(float) when reading into Pandas.

CloudWatch Logs

Your Lambda writes all logger.info() and logger.error() output to CloudWatch. This is the primary debugging tool once your Lambda is deployed.

What Command
Live tail (follow new logs) aws logs tail /aws/lambda/adflow-YOURID-worker --follow
Search for "Batch complete" aws logs filter-log-events --log-group-name /aws/lambda/adflow-YOURID-worker --filter-pattern "Batch complete"
Search for errors aws logs filter-log-events --log-group-name /aws/lambda/adflow-YOURID-worker --filter-pattern "ERROR"
AWS Console path CloudWatch > Log groups > /aws/lambda/adflow-YOURID-worker

Logging Best Practices

import logging
import time

logger = logging.getLogger()
logger.setLevel(logging.INFO)

# Measure and log performance
start = time.perf_counter()
# ... do work ...
elapsed_ms = (time.perf_counter() - start) * 1000
logger.info("Processed %s in %.1f ms", opportunity_id, elapsed_ms)

# Log structured data
logger.info(
    "Batch complete: %d/%d succeeded in %.1f ms | failures=%d",
    success_count, total_count, elapsed_ms, failure_count,
)

# NEVER use print() in Lambda - use logger instead
# print() works but does not include timestamp or log level

Cost Safety Reminders

  • Always run sam delete when you are finished for the day
  • DynamoDB uses PAY_PER_REQUEST billing - you pay only for what you use
  • Lambda has a generous free tier (1M requests/month) - you will not exceed it
  • SQS free tier covers 1M requests/month - cleanup to avoid message retention costs
  • Check your AWS billing dashboard if anything looks unexpected
