Week 1 Overview (Tue Jan 27, Thu Jan 29)

This week establishes core course tooling: Git (branches, merges, pull requests), GitHub workflow, and AWS onboarding (IAM users/roles, AWS CLI, budget alarms). You will complete a guided merge-conflict exercise and publish a personal GitHub Pages site.

Tuesday — Git fundamentals

  • Concepts: commits, branches, merges, fast-forward vs merge commits, resolving conflicts.
  • Practice: feature branching and pull requests in GitHub; required reviews.

Thursday — AWS onboarding

  • IAM basics: users vs roles, least privilege, access keys.
  • AWS CLI configuration: profiles, default region, test calls.
  • Cost safety: budgets/alerts; tagging; teardown checklists.

AWS Setup Guides

  1. AWS Overview — What is AWS and why we use it
  2. AWS Account Setup — Create account, billing alerts, IAM user
  3. AWS CLI Setup — Install and configure AWS CLI

Readings

  • Git — Branches in a Nutshell
  • Git — git merge & conflicts
  • GitHub — About pull requests
  • Resolve merge conflicts
  • AWS IAM — Users, groups, roles
  • AWS CLI — Configure
  • AWS Budgets — Manage costs

Deliverables (due Sun Feb 1, 11:59 pm)

  1. Git Conflict Kata: complete lab steps and submit your PR link in Canvas.
  2. GitHub Pages: publish a simple page from a branch and submit the public URL.
  3. AWS CLI Verification Assignment: Complete AWS account setup and CLI configuration, then submit output of aws sts get-caller-identity.
