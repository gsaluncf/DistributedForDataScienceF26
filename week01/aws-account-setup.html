AWS Account Setup

In this guide, you'll create your AWS account, set up billing protection, create an IAM user for daily use, and optionally grant your instructor access for troubleshooting.

Time required: 20-30 minutes

Important: Credit Card Required

AWS requires a credit card to create an account. However, you will not be charged if you stay within the Free Tier limits. We'll set up billing alerts immediately to protect you from unexpected charges.

Part 1: Create Your AWS Account

Step 1: Sign Up for AWS

  1. Go to https://aws.amazon.com/free/
  2. Click "Create a Free Account"
  3. Enter your email address (use your NCF email: yourname@ncf.edu)
  4. Choose an AWS account name (e.g., "yourname-ncf")
  5. Click "Verify email address" and check your email for the verification code

Step 2: Complete Account Information

  1. Create a strong root password (save this in a password manager!)
  2. Enter your contact information
  3. Choose "Personal" account type
  4. Enter your credit card information
    • AWS will charge $1 temporarily to verify the card (refunded immediately)
  5. Complete phone verification
  6. Choose the "Basic Support - Free" plan

Account Created!

You should now see the AWS Management Console. Let's immediately set up billing protection.

Part 2: Set Up Billing Alerts (CRITICAL!)

This is the most important step to prevent unexpected charges. We'll create email alerts that notify you if your spending exceeds certain thresholds.

Step 1: Enable Billing Alerts

  1. In the AWS Console, click your account name (top right) → "Account"
  2. Scroll down to "Billing preferences"
  3. Click "Edit"
  4. Check these boxes:
    • ✓ "Receive Free Tier Usage Alerts"
    • ✓ "Receive Billing Alerts"
  5. Enter your email address
  6. Click "Update"

Step 2: Create CloudWatch Billing Alarms

Important: Switch your AWS Console to the us-east-1 (N. Virginia) region before creating billing alarms. Billing metrics are only available in this region. Check the region selector in the top-right corner and select "US East (N. Virginia)" before proceeding.

  1. In the search bar at the top, type "CloudWatch" and select it
  2. In the left sidebar, click "Alarms" → "All alarms"
  3. Click "Create alarm"
  4. Click "Select metric"
  5. Click "Billing" → "Total Estimated Charge"
  6. Check the box next to "USD"
  7. Click "Select metric"

Step 3: Configure the Alarm

  1. Under "Conditions":
    • Threshold type: "Static"
    • Whenever EstimatedCharges is: "Greater"
    • than: 1 (start with $1)
  2. Click "Next"
  3. Under "Notification":
    • Select "Create new topic"
    • Topic name: "billing-alerts"
    • Email: Your NCF email
  4. Click "Create topic"
  5. Click "Next"
  6. Alarm name: "billing-alert-1-usd"
  7. Click "Next" → "Create alarm"

Check Your Email!

AWS will send you a subscription confirmation email. You MUST click the confirmation link for the alerts to work!

Step 4: Create Additional Alarms (Recommended)

Repeat the alarm creation process for these thresholds:

  • $5 - Warning level
  • $10 - Serious concern

Name them billing-alert-5-usd and billing-alert-10-usd.

Part 3: Create an IAM User

Your root account has full access to everything. For security, we'll create an IAM user for daily use.

Step 1: Access IAM

  1. In the search bar, type "IAM" and select "IAM"
  2. In the left sidebar, click "Users"
  3. Click "Create user"

Step 2: Configure User Details

  1. User name: your-name-admin (e.g., "john-admin")
  2. Check "Provide user access to the AWS Management Console"
  3. Choose "I want to create an IAM user"
  4. Choose "Custom password" and enter a strong password
  5. Uncheck "Users must create a new password at next sign-in"
  6. Click "Next"

Step 3: Set Permissions

  1. Select "Attach policies directly"
  2. Search for and check "AdministratorAccess"
  3. Click "Next"
  4. Click "Create user"

Save Your Sign-In URL!

AWS will show you a Console sign-in URL that looks like:

https://123456789012.signin.aws.amazon.com/console

Save this URL! You'll use it to sign in as your IAM user (not the root account).

Part 4: Create Access Keys for CLI

To use AWS from the command line, you'll need access keys. We'll create these in the next guide, but here's what you need to know:

Step 1: Sign In as Your IAM User

  1. Sign out of the AWS Console (if signed in as root)
  2. Go to your Console sign-in URL (saved from Part 3)
  3. Sign in with your IAM username and password

Step 2: Create Access Keys

  1. Click your username (top right) → "Security credentials"
  2. Scroll down to "Access keys"
  3. Click "Create access key"
  4. Select "Command Line Interface (CLI)"
  5. Check the confirmation box
  6. Click "Next"
  7. Description: "CLI access for distributed systems course"
  8. Click "Create access key"

CRITICAL: Save Your Keys!

AWS will show you:

  • Access key ID (starts with "AKIA...")
  • Secret access key (long random string)

Download the CSV file or copy both keys to a secure location. You'll need these for the CLI setup!

WARNING: You can only see the secret key ONCE. If you lose it, you'll need to create new keys.

Part 5: Grant Instructor Access (Optional)

Optional: Allow Instructor to Help

If you get stuck with AWS issues, you can grant your instructor (gsalu@ncf.edu) read-only access to help troubleshoot. This is completely optional.

Step 1: Create a Read-Only User for Instructor

  1. In IAM, click "Users" → "Create user"
  2. User name: instructor-readonly
  3. Check "Provide user access to the AWS Management Console"
  4. Choose "I want to create an IAM user"
  5. Choose "Autogenerated password"
  6. Check "Users must create a new password at next sign-in"
  7. Click "Next"

Step 2: Set Read-Only Permissions

  1. Select "Attach policies directly"
  2. Search for and check "ReadOnlyAccess"
  3. Click "Next" → "Create user"

Step 3: Share Credentials with Instructor

  1. Download the CSV file with the sign-in details
  2. Email it to gsalu@ncf.edu with subject: "AWS Read-Only Access - [Your Name]"
  3. Include your Account ID (12-digit number) in the email

What Can the Instructor See?

With ReadOnlyAccess, the instructor can:

  • ✓ View your AWS resources (S3 buckets, Lambda functions, etc.)
  • ✓ See billing information to help diagnose cost issues
  • ✓ Check configuration settings
  • ✗ Cannot create, modify, or delete anything
  • ✗ Cannot see your access keys or passwords
