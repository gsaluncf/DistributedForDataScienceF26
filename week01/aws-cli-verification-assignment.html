Assignment: AWS CLI Verification

Prove that you have successfully set up your AWS account and configured the AWS CLI by submitting the output of a verification command.

Points: 100 points

Due: Sunday, February 1, 11:59 PM

Prerequisites

Before completing this assignment, you must have:

  • ✓ Created your AWS account
  • ✓ Set up billing alerts (and confirmed the SNS subscription email!)
  • ✓ Created an IAM user with AdministratorAccess
  • ✓ Created access keys for your IAM user
  • ✓ Installed AWS CLI on your computer
  • ✓ Configured AWS CLI with aws configure

Instructions

Step 1: Run the Verification Command

Open your terminal (PowerShell on Windows, Terminal on Mac/Linux) and run:

aws sts get-caller-identity

Step 2: Copy the Output

You should see JSON output that looks like this:

{
    "UserId": "AIDAXXXXXXXXXXXXXXXXX",
    "Account": "123456789012",
    "Arn": "arn:aws:iam::123456789012:user/your-name-admin"
}

Copy the entire output from your terminal.

Step 3: Submit to Canvas

Paste the output into the text submission box below.

What We're Checking

This command verifies that:

  • ✓ AWS CLI is installed correctly
  • ✓ Your credentials are configured properly
  • ✓ You can authenticate with AWS
  • ✓ You're using an IAM user (not root account)

The output shows your User ID, Account ID, and ARN (Amazon Resource Name) which includes your IAM username.

Troubleshooting

Error: "Unable to locate credentials"

Run aws configure again and enter your access keys.

Error: "aws: command not found"

AWS CLI is not installed or not in your PATH. Reinstall AWS CLI and restart your terminal.

Error: "The security token included in the request is invalid"

Your access keys are incorrect. Double-check that you copied them correctly from the AWS console.

ARN shows "root" instead of IAM user

You're using root account credentials. Go back to the AWS Account Setup guide and create an IAM user with access keys.
