AWS CLI Setup & Verification

Install the AWS Command Line Interface (CLI) and verify your setup. This is your assignment submission!

What is AWS CLI?

The AWS CLI is a command-line tool that lets you interact with AWS services from your terminal. It's essential for automation, scripting, and efficient cloud management.

Prerequisites

Before starting, make sure you have completed the AWS Account Setup guide and have:

  • ✓ Created your AWS account
  • ✓ Set up billing alerts
  • ✓ Created an IAM user with AdministratorAccess
  • ✓ Created and saved your access keys (Access Key ID and Secret Access Key)

Step 1: Install AWS CLI

Windows

  1. Open PowerShell as Administrator
  2. Run this command to download and install AWS CLI: 
    msiexec.exe /i https://awscli.amazonaws.com/AWSCLIV2-2.33.4.msi
  3. Follow the installation wizard (use default settings)
  4. Close and reopen PowerShell

Alternative: Manual Download

If the command above doesn't work, you can manually download the installer:

  • AWS CLI MSI Installer for Windows (v2.33.4)

Run the downloaded file and follow the installation wizard.

macOS

  1. Open Terminal
  2. Download and install AWS CLI: 
    curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"
sudo installer -pkg AWSCLIV2.pkg -target /

Linux

curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

Step 2: Verify Installation

Open a new terminal and run:

aws --version

You should see output like: aws-cli/2.x.x Python/3.x.x

Step 3: Configure AWS CLI

Now we'll configure the AWS CLI with your access keys from the account setup.

Run AWS Configure

In your terminal, run:

aws configure

Enter Your Credentials

You'll be prompted for four values. Enter them as follows:

AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
Default region name [None]: us-east-1
Default output format [None]: json

Important Notes

  • Access Key ID: Paste your actual Access Key ID (starts with "AKIA...")
  • Secret Access Key: Paste your actual Secret Access Key (long random string)
  • Region: Use us-east-1 (most AWS services are available here)
  • Output format: Use json (easiest to read)

What Just Happened?

The aws configure command created two files:

  • Credentials file: Stores your access keys
    • Windows: C:\Users\YourName\.aws\credentials
    • Mac/Linux: ~/.aws/credentials
  • Config file: Stores your default region and output format
    • Windows: C:\Users\YourName\.aws\config
    • Mac/Linux: ~/.aws/config

Step 4: Test Your Configuration (ASSIGNMENT)

Verify Your Identity

Run this command to verify your AWS CLI is configured correctly:

aws sts get-caller-identity

You should see output like:

{
    "UserId": "AIDAXXXXXXXXXXXXXXXXX",
    "Account": "123456789012",
    "Arn": "arn:aws:iam::123456789012:user/your-name-admin"
}

Success!

If you see your account information with your IAM username, your AWS CLI is configured correctly!

Test S3 Access

Try listing your S3 buckets (should be empty for now):

aws s3 ls

If you see no errors (even if the output is empty), you're ready to go!

Troubleshooting

Error: "Unable to locate credentials"

This means your credentials weren't saved correctly. Try running aws configure again and make sure you paste your keys correctly.

Error: "The security token included in the request is invalid"

Your access keys may be incorrect. Double-check that you:

  • Copied the entire Access Key ID (starts with "AKIA")
  • Copied the entire Secret Access Key
  • Didn't include any extra spaces or quotes

Error: "aws: command not found"

The AWS CLI is not in your PATH. Try:

  • Close and reopen your terminal
  • On Windows, make sure you opened a new PowerShell window after installation
  • Reinstall AWS CLI if the problem persists
