Kafka Messaging with Confluent Cloud

Event streaming, message brokers, producer/consumer patterns, and consumer groups — using a managed Kafka cluster with no local infrastructure.
This Week

Tuesday: Lecture — where brokers live in the stack, what Kafka solves, core concepts (topics, partitions, offsets, consumer groups), and the complexities streaming introduces.

Thursday: Lab — sign up for Confluent Cloud, produce and consume sensor telemetry with Python, observe consumer group rebalancing, and build a two-stage streaming pipeline.

Part 1 Materials

Lecture
Event Streaming and Message Brokers
Request/response vs event-driven architecture, Kafka use cases, topics and partitions, consumer groups, delivery guarantees, and where Kafka fits in the modern data stack.
Lab
Producer/Consumer with Confluent Cloud
Set up a managed Kafka cluster, stream greenhouse sensor data with Python, scale consumers horizontally, and build a real-time alert pipeline.
