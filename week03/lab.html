Why requirements.txt Lists boto3

Important clarification: boto3 is already included in the AWS Lambda Python runtime, so technically you don't need to install it.

However, we list it in requirements.txt for two reasons:

1. Explicit dependencies: Makes it clear what libraries the code uses
2. Local testing: If you run the code locally (outside Lambda), you'll need boto3 installed
3. Version pinning: Ensures consistent behavior across environments

In production projects, you'd only list third-party libraries NOT included in Lambda (like requests, pandas, etc.).

Step 2.5: Review the Lambda Handler (handler.py)

Before building and deploying, take a moment to read the Lambda handler you'll deploy. In this repo it's located at src/handler.py. If you're using the class template from GitHub Classroom, you'll get the same file from the assignment link: GitHub Classroom Assignment.

import json
import os
import boto3
from decimal import Decimal

# Initialize DynamoDB resource
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table(os.environ.get('TABLE_NAME', 'items'))


def lambda_handler(event, context):
    """
    Main Lambda handler for API requests.
    Routes requests based on HTTP method.
    """
    # Get HTTP method from event (supports both v1.0 and v2.0 formats)
    http_method = event.get('httpMethod') or event.get('requestContext', {}).get('http', {}).get('method')
    
    if http_method == 'GET':
        return handle_get(event)
    elif http_method == 'POST':
        return handle_post(event)
    else:
        return {
            'statusCode': 405,
            'body': json.dumps({'error': 'Method not allowed'})
        }


def handle_get(event):
    """
    Handle GET requests to retrieve an item by ID.
    """
    # Extract item ID from path parameters
    item_id = event.get('pathParameters', {}).get('id')
    
    if not item_id:
        return {
            'statusCode': 400,
            'body': json.dumps({'error': 'Missing id parameter'})
        }
    
    try:
        # Get item from DynamoDB
        response = table.get_item(Key={'id': item_id})
        
        if 'Item' in response:
            # Convert Decimal to float for JSON serialization
            item = convert_decimals(response['Item'])
            return {
                'statusCode': 200,
                'body': json.dumps(item)
            }
        else:
            return {
                'statusCode': 404,
                'body': json.dumps({'error': 'Item not found'})
            }
    except Exception as e:
        return {
            'statusCode': 500,
            'body': json.dumps({'error': str(e)})
        }


def handle_post(event):
    """
    Handle POST requests to create a new item.
    """
    try:
        # Parse request body
        body = json.loads(event.get('body', '{}'))
        
        # Validate required fields
        item_id = body.get('id')
        if not item_id:
            return {
                'statusCode': 400,
                'body': json.dumps({'error': 'Missing id in request body'})
            }
        
        # Convert floats to Decimal for DynamoDB
        item = convert_floats(body)
        
        # Store item in DynamoDB
        table.put_item(Item=item)
        
        return {
            'statusCode': 201,
            'body': json.dumps({'message': 'Item created', 'id': item_id})
        }
    except Exception as e:
        return {
            'statusCode': 500,
            'body': json.dumps({'error': str(e)})
        }


def convert_floats(obj):
    """
    Recursively convert float values to Decimal for DynamoDB storage.
    """
    if isinstance(obj, list):
        return [convert_floats(i) for i in obj]
    elif isinstance(obj, dict):
        return {k: convert_floats(v) for k, v in obj.items()}
    elif isinstance(obj, float):
        return Decimal(str(obj))
    else:
        return obj


def convert_decimals(obj):
    """
    Recursively convert Decimal values to float for JSON serialization.
    """
    if isinstance(obj, list):
        return [convert_decimals(i) for i in obj]
    elif isinstance(obj, dict):
        return {k: convert_decimals(v) for k, v in obj.items()}
    elif isinstance(obj, Decimal):
        return float(obj)
    else:
        return obj

What This Handler Does

1. Routes requests by HTTP method in lambda_handler() (supports API Gateway v1 and v2 events).
2. GET /items/{id}: extracts id from pathParameters and retrieves the item from DynamoDB.
3. POST /items: validates that id exists in the request body, converts floats to Decimal, and stores the item.
4. Uses environment variable TABLE_NAME (set by SAM) to choose the DynamoDB table.
5. Serialization helpers: convert_floats() (float→Decimal for writes) and convert_decimals() (Decimal→float for JSON responses).
6. Returns appropriate HTTP status codes: 201 on create, 200 on success, 400/404 for bad input/not found, 405 for unsupported methods, and 500 on errors.

Why Decimal Conversion Matters

DynamoDB stores numbers precisely. Using Python float can cause TypeError: Float types are not supported. The handler converts floats → Decimal before writes, and converts them back to floats when returning JSON.

What If Deployment Fails?

If deployment fails, SAM will automatically roll back changes. Common issues:

Insufficient permissions: Your AWS credentials need CloudFormation, Lambda, API Gateway, DynamoDB, and IAM permissions
Stack name already exists: Choose a different stack name or delete the existing stack first
Resource limits: Check your AWS account limits (rare for new accounts)

SAM will show detailed error messages in the CloudFormation events. Check the AWS Console \u2192 CloudFormation \u2192 Events tab for details.

Monitoring Costs During the Lab

Now that your API is deployed, let's test both endpoints to make sure everything works.

Option 1: Web UI Tester (Easiest)

We've provided a local HTML file that lets you test your API directly from your browser—no command-line tools needed! This is a standalone testing tool that you open locally on your computer.

Step-by-Step Instructions:

  1. Locate the tester file: In your project folder (where you cloned the repository), find local_tester.html. It's in the same directory as your template.yaml file.
  2. Open in your browser:
    • Windows: Right-click local_tester.html → "Open with" → Choose your browser (Chrome, Edge, Firefox)
    • macOS: Right-click local_tester.html → "Open With" → Choose your browser
    • Alternative: Drag and drop local_tester.html into an open browser window
  3. Get your API URL: From Step 4 deployment outputs, copy the ApiUrl value. It looks like: 
    https://abc123xyz.execute-api.us-east-1.amazonaws.com
    Important: Copy it exactly—no trailing slash at the end!
  4. Paste your API URL: In the tester page, paste your API URL into the "API URL" field at the top.
  5. Test POST /items: The page has a pre-filled JSON body like: 
    {
  "id": "item-001",
  "name": "Laptop",
  "price": 999.99
}
    Click the "POST - Create Item" button. You should see:
    • Status: 201 (Created)
    • Response showing the item was created successfully
  6. Test GET /items/{id}: In the "Item ID" field, enter item-001 (or whatever ID you used in POST). Click "GET - Retrieve Item". You should see:
    • Status: 200 (OK)
    • Response: The full item data you stored
Common Issues and Solutions
  • CORS error: Make sure you copied the API URL correctly and that your SAM template includes the CORS configuration (it does by default).
  • 404 Not Found: Double-check your API URL. Make sure you're using the ApiUrl output from sam deploy, not the Lambda function name.
  • Trailing slash issue: The URL should be ...amazonaws.com, NOT ...amazonaws.com/
  • Item not found (GET): Make sure you created the item with POST first, and you're using the exact same ID.

How it works: The tester is a simple HTML page with JavaScript that makes HTTP requests to your API. The SAM template includes CORS (Cross-Origin Resource Sharing) configuration, which allows browsers to make requests from file:// URLs to your API. Without CORS, browsers would block these requests for security reasons.

Behind the scenes: When you click the buttons, the tester calls:
• POST {ApiUrl}/items - Creates a new item in DynamoDB
• GET {ApiUrl}/items/{id} - Retrieves an item from DynamoDB

Understanding CORS

CORS is configured in template.yaml under the Globals section:

Globals:
  Api:
    Cors:
      AllowMethods: "'GET,POST,OPTIONS'"
      AllowOrigin: "'*'"

AllowOrigin: "'*'" means any website can call your API. In production, you'd restrict this to your specific domain (e.g., "'https://myapp.com'").

Without CORS: Browser-based testing would fail with errors like "blocked by CORS policy". Command-line tools (curl, PowerShell) are not affected by CORS.

Step 7: Clean Up Resources

When you're done testing, it's important to delete your resources to avoid ongoing charges.

Important: Clean Up to Avoid Charges

While this lab uses minimal resources, it's good practice to delete everything when you're done. DynamoDB on-demand mode charges per request, and Lambda charges per invocation.

Delete the Stack

SAM makes cleanup easy with a single command:

sam delete --stack-name week3-yourname

SAM will ask for confirmation. Type y to proceed.

What Gets Deleted

Resource What Happens
DynamoDB Table Deleted (all data is lost)
Lambda Function Deleted (code and configuration removed)
API Gateway Deleted (API URL becomes invalid)
IAM Roles Deleted (permissions removed)
CloudWatch Logs Retained (you can manually delete if desired)

Verify Deletion

After deletion completes, verify in the AWS Console:

Service What to Check
CloudFormation Stack should show DELETE_COMPLETE status
DynamoDB Table should no longer appear in tables list
Lambda Function should no longer appear in functions list

Troubleshooting

Problem Solution
sam build fails with Python not found Ensure Python 3.14 is installed and in your PATH, and make sure your virtual environment is activated before running sam build.
