Benchmarking DataFrame Workloads

Reproduce the same ETL in pandas and Polars, record runtimes and memory, and assert thresholds to enable autograding. Use Parquet for IO where possible.

Dataset

Use the provided synthetic CSV (~200MB) from the assignment repo (/data/synthetic.csv). You may convert to Parquet for faster repeated runs.

Tasks

  1. Load data; ensure column dtypes are optimal (categoricals/ints where applicable).
  2. Compute: groupby aggregations (mean/sum), filtered joins, windowed rank, and a derived column.
  3. Write result to Parquet with snappy compression.

Benchmark protocol

  • Warm-up once, then record the best of 3 runs for each library.
  • Measure wall time and peak RSS (process memory) if possible.
  • Save timings to a small JSON (pandas_time, polars_time, pandas_mem_mb, polars_mem_mb).

Autograding assertions

  • Results equality check: outputs must be identical (same row count and key metrics within tolerance).
  • Performance check: polars_time ≤ 0.75 × pandas_time on the benchmark machine (set as a soft threshold in tests).

What to submit

  • Notebook (.ipynb) with both implementations and timing cells.
  • Exported Parquet output (first 100 rows) and timing JSON.
  • Short memo (<=300 words): when to switch to Polars; caveats.
