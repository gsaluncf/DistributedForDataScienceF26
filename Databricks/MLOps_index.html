MLOps — Model Lifecycle on Databricks

Feature engineering, model training, holdout evaluation, MLflow tracking, and Model Registry promotion — using a social media recommendation system as the running example.
This Week

Discussion: What MLOps is, why it exists, and how it connects to everything built this semester — from raw Kafka events to a versioned, registered model ready for deployment.

Lab: Run the complete 5-stage pipeline on Databricks using real Bluesky social network data. Train two competing models, evaluate both with a holdout metric, compare runs in the MLflow UI, and register the winner in the Model Registry.

Part 2 Pipeline at a Glance

Five artifacts are produced, each a permanent record that can be inspected, reproduced, or handed off to a downstream system independently of the notebook that created it.

Stage 1
Feature Table
Weighted interaction score per user-account pair. Stored as a Delta table. Input to both models.
Stage 2
ALS Model
Matrix factorization via the implicit library. Serialized as an MLflow pyfunc artifact. Top-10 recs per user in Delta.
Stage 3
TF Embedding
Keras model trained on positive and negative pairs. Also in MLflow. Top-10 recs in Delta.
Stage 4
Evaluation
Hit Rate @ 10 via holdout. Logged as a metric to each MLflow run — queryable and permanent.
Stage 5
Registry
Winning model promoted to the MLflow Model Registry with a version number. Ready for deployment.

Part 3 Notebook

The lab runs in a single Databricks notebook. Import it into your workspace or clone the course repo. Each section below corresponds to one stage in the pipeline — run them in order.

Lab 1 Notebook — nb_mlops.py

Installs implicit (ALS library) via %pip install at the top, which triggers an automatic Python session restart — this is expected. After the restart, run all cells from the top in order. Completes in ~4–6 minutes on serverless compute.

Lab 2 Notebook — nb_mlops_tf.py

Requires Lab 1 to be complete first — it reads the feature table, holdout set, and ALS recommendation table created in Lab 1. Installs tensorflow>=2.16 at the top, which also triggers a Python session restart. After the restart, run all cells from the top in order. Completes in ~8–12 minutes on serverless compute.
