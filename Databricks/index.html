Labs Hands-On Notebooks

After setting up your workspace and cloning the course repo (see guide Section 6.2), you will find these notebooks at Databricks/student_notebooks/. Work through them in order.

Notebook 1 nb1_spark_intro.py

Spark Intro & DataFrames. Load tables, apply transformations, trigger actions, and see the execution plan (DAG). Covers lazy evaluation, column pruning, shuffles, and broadcast joins.

Notebook 2 nb2_file_formats.py

File Formats & Table Formats. Write the same dataset in five formats (CSV, Parquet, Avro, Delta Lake, Iceberg) and compare schema safety, compression, and query speed. Explore Delta Lake time travel, schema enforcement, and Unity Catalog Volumes.

Notebook 3 nb3_sql.py

SQL on Spark. Everything you know from SQL, but distributed. Use %sql magic cells, spark.sql() for Python integration, temp views, CTEs, window functions (RANK, running totals), and CREATE TABLE AS SELECT.

Notebook 4 nb4_dbsql.py

DBSQL & the Modern Stack. Move beyond PySpark to SQL-first workflows. Covers MPP architecture, side-by-side DBSQL vs PySpark comparisons, serverless compute, partitioning, Query Profile analysis, and the evolving ecosystem (Lakeflow, Genie, Databricks Apps, AI Agents).

Notebook 5a nb5_bronze.py

Bronze Layer: Raw Ingestion. Read a custodian's messy Excel report from a Unity Catalog Volume, convert both tabs to Spark DataFrames, and persist them as raw Delta tables with ingestion metadata. No cleaning, just capture.

Notebook 5b nb5_silver.py

Silver Layer: Clean and Conform. Normalize tickers, strip dollar signs, parse mixed date formats, deduplicate, and cast types. Write clean Delta tables and versioned CSVs back to the Volume.

Notebook 5c nb5_gold.py

Gold Layer: Portfolio Valuation. Join cleaned holdings with cleaned prices to compute market value, unrealized P&L, and portfolio weight per position. Build a sector allocation summary. Wire all three notebooks into a Databricks Workflow.

Notebook 6 nb6_declarative.py

Declarative Pipeline (Lakeflow DLT). A self-contained notebook that declares tables as contracts using @dlt.table() decorators with inline sample data. Data quality rules via @dlt.expect_or_drop() are part of the contract. Databricks resolves the DAG during code loading. Run as a Pipeline (not a regular notebook).

Getting the notebooks

Recommended: Clone the course repository directly into Databricks using Git integration (see guide Section 6.2 for step-by-step instructions with screenshots). This gives you all notebooks and makes it easy to pull updates.
Alternatively: Download individual notebook files from the GitHub repo and import them via Workspace → Import → URL.

Overview What You Will Learn

This module bridges your existing database knowledge with distributed analytics at scale:

  • Distributed query execution — how Spark breaks queries into tasks and runs them across workers in parallel (DAG, shuffles, stages)
  • DataFrames and lazy evaluation — Spark's API for transformations on distributed data
  • File and table formats — CSV, Parquet, Delta Lake, and why format choice matters for performance and reliability
  • Unity Catalog — centralized governance with a three-level namespace (catalog.schema.table)
  • SQL at scale — everything you know from databases, now distributed
  • DBSQL and the modern stack — the industry shift from PySpark to SQL-first workflows, serverless compute, and platform-managed infrastructure
  • MPP architecture — coordinator + workers: the same distributed pattern you see in every system this semester
The Big Idea

Databricks shows where the industry is headed: you write SQL, and the platform handles parallelism, fault tolerance, and optimization. But understanding the distributed system underneath — partitions, shuffles, execution plans — is what separates someone who can use these tools from someone who can debug and optimize them.
