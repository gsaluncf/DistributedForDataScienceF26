Install AWS CLI v2

Follow the steps for your OS. Verify with aws --version and aws sts get-caller-identity. Screenshots are indicated with placeholders—capture them during class for Canvas.

Windows 10/11 (PowerShell)

  1. Download installer: AWSCLIV2.msi.
  2. Run the MSI (Next → Next). Accept defaults.
  3. Open Windows PowerShell and run: 
    aws --version
aws configure --profile student
# When prompted, paste your Access Key ID, Secret, default region (e.g., us-east-1), and output format (json)
  4. Verify identity (after creating your IAM user and keys): 
    aws sts get-caller-identity --profile student

macOS (Intel/Apple Silicon)

  1. Using Homebrew (recommended): 
    brew install awscli
  2. Or download the PKG from official docs.
  3. Verify and configure: 
    aws --version
aws configure --profile student

Linux

  1. Install via package manager or bundle: 
    curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
aws --version
  2. Configure profile: 
    aws configure --profile student

Troubleshooting

  • Command not found: Close and reopen terminal; ensure AWS CLI is on PATH.
  • AccessDenied: Use the correct --profile student or re-run aws configure --profile student.
  • Wrong region: Set default region during aws configure (e.g., us-east-1).

Key Terms

  • AWS CLI: A command-line tool to interact with AWS services.
  • Profile: A named set of credentials and default settings used by AWS CLI, e.g., --profile student.
  • Region: A geographic area where AWS hosts resources (e.g., us-east-1).
