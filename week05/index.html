Week 5 Overview (Tue Feb 24, Thu Feb 26)

As a class, we will build a horizontally scalable "Distributed Cipher Cracker" using AWS Lambda, SQS, DynamoDB, and S3. Each group contributes workers running in their own AWS accounts. Emphasis on concurrency controls, idempotency, and cost safety.

Tuesday — System design & scaffolding

  • Agree on task schema (SQS message shape) and result schema (DynamoDB item).
  • Define keyspace/wordlist partitioning and retry/visibility-timeout strategy.
  • Create starter repositories from a common template.

Thursday — Implementation & load test

  • Implement Lambda worker with concurrency limit; add structured logs.
  • Run a controlled load; monitor SQS depth, Lambda invocations, DynamoDB throttles.
  • Summarize findings and cost considerations.

Readings

  • Lambda scaling & concurrency
  • SQS visibility timeout
  • DynamoDB TTL
  • S3 lifecycle management

Deliverables (due Sun Mar 1, 11:59 pm)

  1. Group repo URL (template-based) with worker implementation.
  2. Screenshot: CloudWatch metrics (invocations/concurrency) and SQS queue depth during test.
  3. Screenshot: DynamoDB table with results (with TTL configured); brief reflection (<= 300 words) on scaling/cost.
