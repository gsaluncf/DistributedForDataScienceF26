Design Exercise

Write a 1–2 page critique comparing two approaches to scale a workload. Support arguments with concepts (Amdahl/Gustafson, back-pressure, idempotency) and map to AWS services.

Scenario

You receive CSV files (~5 GB/day) uploaded sporadically. Within 5 minutes of arrival, you must parse, validate, and store records into a queryable store; a downstream system reads aggregated results each hour. Failures must not lose data. Costs should scale with use.

Approach A (Event-driven serverless)

  • S3 put event → EventBridge → Step Functions
  • Lambda for chunk parse; SQS for fan-out/back-pressure
  • Write Parquet to S3; manifest in DynamoDB for idempotency

Approach B (Always-on cluster)

  • EC2 Auto Scaling Group or EMR cluster
  • Daemon watches S3, triggers batch job
  • Writes to S3 (Parquet) and notifies downstream

Prompt

  1. Compare throughput, latency, and tail behavior at peak.
  2. Discuss idempotency and failure handling (DLQs, retries).
  3. Evaluate cost (idle vs per-request), and operational burden.
  4. Recommend one approach; list 2 risks and mitigations.

Submission

  • Upload PDF (1–2 pages). Include citations to docs as needed.
  • Optional diagram (1 page max) not counted against length.
